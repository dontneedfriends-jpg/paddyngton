# 📖 Paddyngton

### Кроссплатформенный редактор для написания книг

---

*"Зачем писать книги в Word, если можно страдать с Tauri + React и получить 7 тем оформления?"*

---

## Возможности

### ✍️ Markdown Editor
- **CodeMirror 6** — подсветка синтаксиса для избранных
- **14 кнопок форматирования** — от зачёркивания до `<sup>верхнего индекса</sup>`. Зачем? Никто не знает. Но они есть
- **Line numbers** — для тех, кто всегда хотел знать, на какой строке допустил ошибку
- **Auto-save** — потому что Ctrl+S — это для слабаков

### 🗂️ Context System (Персонажи, Места, Даты, Предметы)

| Тип | Зачем | Пример |
|------|------|--------|
| character | Кто? | "Герой, который забывает ключи в 7-й главе" |
| place | Где? | "Квартира, которая никогда не убирается" |
| date | Когда? | "Третий день без интернета" |
| item | Что? | "Загадочная склянка с подписью 'не пить'" |

Каждая запись содержит **details** (поля зависят от типа), **relations**, **group** и **notes**. Details для персонажа: Age, Gender, Occupation, Personality, Appearance, Background, Motivation, Flaws, Strengths, Skills, Status, FirstAppearance.

### 🕸️ Mind Map (Interaction Map)

Визуальная карта отношений между персонажами. Это как "семейная драма" на Thanksgiving, только без тёти Мэйбл.

- Таскайте персонажей по холсту (canvas-relative координаты — не спрашивайте почему)
- Рисуйте линии между ними
- Линии раскрашиваются по типу отношения:
  - **ally** (зелёный #38a169) — союзник
  - **enemy** (красный #e53e3e) — враг
  - **family** (синий #3182ce) — семья
  - **neutral** (серый #718096) — нейтрально
  - **romantic** (жёлтый #d69e2e) — романтика
  - **rival** (фиолетовый #805ad5) — соперник

### 📚 Wiki View

Просмотр деталей любого персонажа/места/предмета. Редактируйте свойства, смотрите связи, удаляйте отношения. Всё, чтобы запутаться в собственных персонажах.

### ⏱️ Timeline

События с датами, цветом, заметками и linked characters. Сортировка по дате. Потому что хронология — это важно.

### 📝 Notes

Заметки. Просто заметки. Без категорий. Без ограничений.

### 🌍 World Building

Записи о мире с категориями. Сетка из карточек. Потому что мир больше персонажей.

### 📋 Kanban Board

Столбцы с карточками. "Идеи", "В работе", "Готово". Революционно.

### 🎨 7 Themes + Light/Dark

| Theme | Описание |
|-------|---------|
| **Galaxy** | Фиолетовый. Космос, но дешевле |
| **Aurora** | Зелёный. Полярное сияние без поездки в Норвегию |
| **Forest** | Коричневый. Природа |
| **Obsidian** | Чёрный. Для тёмных романов |
| **Neon** | Неоновый. Как вывеска бара в 80-х |
| **Retro** | Как старый телевизор |
| **Monochrome** | Оттенки серого. Для тех, кто не может определиться |

Каждый с variantami light/dark. 14 вариантов оформления — хватит на две книги.

### 🔍 Поиск

Ctrl+Shift+F — ищите по всем главам. Результаты с подсветкой. Потому что grep — слишком сложно.

### 📸 Версионирование

- **Save snapshot** — сохраняйте версии с меткой
- **Word count, char count, chapter count** — для параноиков
- **Restore** — восстанавливайте предыдущие версии. На случай "а давайте удалим всё"

### 🐻 Bear Import/Export

Импорт/экспорт в формате Bear app (zip-архивы с note-сниппетами). Потому что миграция должна быть больной.

### 🌐 i18n

3 языка: English, Español, Русский. Все UI-строки через `t('key')` — никаких хардкодных строк. Переводы в `/translations/`.

## Структура проекта

```
paddyngton/
├── src/                    # React frontend
│   ├── App.tsx             # ~2100 строк. Не судите строго
│   ├── index.css          # ~900 строк CSS. Темы. Много тем.
│   ├── i18n.tsx          # Интернационализация (3 языка!)
│   ├── types/             # TypeScript интерфейсы (~270 строк)
│   ├── hooks/             # useSettings — настройки в localStorage
│   ├── components/       # Dialogs, Panels
│   └── translations/     # en.json, es.json, ru.json
├── src-tauri/             # Rust backend (~390 строк)
│   └── src/lib.rs        # Tauri commands: window, fonts, snapshots, Bear
├── context-template.json    # Шаблон для .context.json
└── book-template.json    # Шаблон для .book.json
```

## Известные подводные камни

1. **Modal z-index**: `inputDialog` и `confirmDialog` должны рендериться после панелей в JSX
2. **Mind Map drag**: Используйте canvas-relative координаты для плавного перетаскивания
3. **SVG z-index**: Узлы mind map (z-index: 2) должны быть над SVG-линиями (z-index: 1)
4. **State-based relation selector**: Не используйте inputDialog — используйте inline dropdown
5. **Translation keys**: Все UI-тексты через `t('key')`

|--------|----------|
| Ctrl+N | Новая книга |
| Ctrl+O | Открыть папку |
